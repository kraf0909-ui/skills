# 速查手册（Cheatsheet）

## 系统1 vs 系统2 快速判断

| 特征 | 系统1（快思考） | 系统2（慢思考） |
|------|----------------|----------------|
| 速度 | 快速 | 缓慢 |
| 努力 | 不费力 | 费力 |
| 意识 | 无意识 | 有意识 |
| 控制 | 自动 | 受控 |
| 典型活动 | 识别人脸、读简单文字、2+2 | 复杂计算、逻辑推理、填税表 |
| 错误类型 | 系统性偏见 | 疲劳导致的疏忽 |

**判断规则**: 如果任务需要集中注意力、会瞳孔放大、无法同时做其他事 → 系统2。

## 主要启发法与偏见速查

| 启发法 | 替换的问题 | 导致的偏见 |
|--------|-----------|-----------|
| 代表性 | "概率是多少？" → "有多像？" | 基础比率忽视、合取谬误 |
| 可得性 | "频率是多少？" → "多容易想起？" | 媒体偏见、个人经历偏差 |
| 锚定 | "数值是多少？" → "从锚调整多少？" | 调整不足、无关数字影响 |
| 情感 | "风险评估？" → "感觉如何？" | 好感度影响判断 |

## 前景理论四重模式

| | 高收益 | 低收益 |
|---|---|---|
| **高概率** | 风险规避（确定收益偏好） | 风险规避 |
| **低概率** | 风险寻求（买彩票） | 风险寻求（买保险） |

**决策规则**:
- 收益域 + 高概率 → 选择确定收益
- 损失域 + 高概率 → 选择赌一把
- 涉及小概率 → 人们对概率不敏感，主要看结果

## 损失厌恶应用规则

```
损失的影响 ≈ 2.0-2.5 × 同等收益的影响

应用:
- 说服他人: 强调不行动的损失，而非行动的收益
- 产品设计: 避免让用户感到"失去"已有功能
- 谈判策略: 将提议框定为避免损失而非获得收益
- 个人决策: 意识到自己对损失的过度反应
```

## 预测改进检查清单

在做重要预测前，问自己：
- [ ] 我是否知道类似案例的基础比率？
- [ ] 我是否受到锚定效应影响？
- [ ] 我的预测是否过于极端？（应向平均值回归）
- [ ] 我是否只看到了支持证据？（WYSIATI）
- [ ] 我是否过于乐观？（计划谬误）
- [ ] 我是否应该使用简单算法替代直觉？
- [ ] 我是否征求了外部意见？

## 常见认知陷阱速查

| 陷阱 | 识别信号 | 防御策略 |
|------|---------|---------|
| 后见之明 | "我早就知道" | 记录事前预测 |
| 过度自信 | "我确定" | 考虑失败场景 |
| 锚定效应 | 第一个数字影响判断 | 主动寻找不同锚点 |
| 可得性偏见 | 最近/生动的例子主导判断 | 查找统计数据 |
| 代表性偏见 | "这很典型" | 检查基础比率 |
| 沉没成本 | "已经投入了这么多" | 只考虑未来收益和成本 |
| 框架效应 | 同一问题的不同表述改变选择 | 尝试多种表述方式 |
| 峰终定律 | 只记住高峰和结局 | 记录整个过程的体验 |

## 专家直觉可信度评估

```
可以相信专家直觉当:
✓ 环境有规律性（如象棋、医学诊断）
✓ 反馈即时且准确
✓ 专家有>10000小时刻意练习
✓ 判断领域有明确规则

不要相信专家直觉当:
✗ 政治预测
✗ 股市预测
✗ 长期经济预测
✗ 复杂社会现象
✗ 专家过于自信
```

## 幸福最大化策略

### 对经验自我（体验效用）:
- 减少日常痛苦（通勤、噪音、压力）
- 增加积极体验频率（社交、运动、自然）
- 注意：持续时间比强度更重要

### 对记忆自我（决策效用）:
- 创造高峰体验
- 确保美好结局
- 注意：过程长度被忽视

### 平衡策略:
- 短期决策: 关注体验自我（不要忍受不必要的痛苦）
- 长期规划: 关注记忆自我（创造值得回忆的时刻）

## 助推（Nudge）设计原则

```
设计选择架构时:

1. 设置好的默认选项
   - 大多数人接受默认
   - 默认应是最有利的选择

2. 简化复杂选择
   - 减少选项数量
   - 提供清晰分类

3. 提供即时反馈
   - 让人们看到行为后果
   - 利用损失厌恶

4. 利用社会规范
   - "大多数人选择..."
   - 同伴压力是强大动力

5. 提高可见性
   - 让正确选择更显眼
   - 让错误选择更困难
```

## 重要数字速记

| 数字 | 含义 |
|------|------|
| 2-2.5x | 损失厌恶系数 |
| 10000小时 | 专家技能习得时间 |
| 7±2 | 工作记忆容量 |
| 4秒 | 系统2反应的典型延迟 |
| 50% | 瞳孔在加3任务中的扩散幅度 |
| 9秒 | 看不见的大猩猩出现时间 |
