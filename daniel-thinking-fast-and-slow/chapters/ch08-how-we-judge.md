# 第8章 我们究竟是如何作出判断的？

## Core Idea

系统1不断监视着大脑内外发生的一切，没有特定意图，也无须付出多少努力，只是对当时的情形作出全方位评估。这些"基本的评估"在直觉性判断中扮演了重要角色，因为人们常会拿它们来替代更难的问题——这也是启发法和偏见研究方法的基本理念。系统1具备跨维度解读价值观的能力（强度匹配），可以回答"如果山姆的身高和智商一样，那么他究竟有多高"这类问题。然而，系统1能解决平均问题，却对总量问题束手无策——一个类别的规模及其所包含的实例数量常常被忽略。

## Frameworks Introduced

- **基本判断（Basic Assessments）**：系统1对当时情形作出的全方位评估，包括事情进展得如何、面临的是威胁还是机遇等。这些评估在直觉性判断中扮演重要角色。
- **强度匹配（Intensity Matching）**：系统1的能力之一，强度的等级在不同领域中都有"匹配"描述。例如，如果罪行是颜色，杀人就应该是深红，颜色比偷窃更深。
- **判断启发法（Judgment Heuristic）**：用简单的判断替代更难的判断。例如，投票者试图对候选人将来的任职表现生成印象，却转而依靠系统2快速自主地作出一种更加简单的判断（如面部特征）。
- **心理物理学（Psychophysics）**：研究物理刺激与心理感受之间关系的学科。系统1可以自主且毫不费力地生成对一组线段的平均长度的印象。

## Key Concepts

- **面部特征与选举预测**：
  - 亚历克斯·托多罗夫的研究：仅凭0.1秒的人脸照片评估，约有70%的参议员、国会议员和地方长官竞选活动的胜出者正是那些在照片评估中获得较高评价的人。
  - 能力评估远比可爱程度的评估对选举结果的预见能力强。
  - 信息贫乏、爱看电视的投票者受面部特征影响的程度约为信息丰富者的3倍。
- **平均长度vs总长度**：
  - 人们在不到一秒钟的时间里可以精确地记下很多线段的平均长度。
  - 但系统1无法为"所有线的总长度是多少"提供建议——总量问题需要系统2介入。
- **拯救鸟类的捐款实验**：
  - 受试者愿意掏钱拯救2000只、20000只和200000只鸟的平均捐款分别是80美元、78美元和88美元。
  - 与鸟的数量没有什么关系——受试者反应的对象是原型（一只无助的小鸟被淹死的画面），几乎完全忽略了数量概念。
- **跨维度强度匹配**：
  - 朱莉4岁时就能阅读——若某人的身高和朱莉的早慧程度一样，那他有多高？（约6-7英尺）
  - 工作收入多高才能与朱莉的阅读能力相匹配？
  - 什么罪行的严重程度可以与朱莉的早慧程度相匹配？
- **思维的发散性**：刻意回答一个问题却引起了另一个问题。例如，听押韵词时即使只要求比较声音，受试者也会同时比较拼写。

## Mental Models

- **系统1的"多任务并行评估"**：系统1任何时候都可以同时进行多种估算，其中有些估算是持续不间断的常规评估（如视觉范围内的立体事物评估）。
- **原型代表模型**：系统1通过原型或一组典型事例来代表不同事物分类。它能解决好平均问题，但对总量问题就束手无策。
- **强度匹配作为跨维度翻译**：系统1能将一个维度上的强度"翻译"到另一个维度上，这种匹配虽然从统计学角度看是错误的，但对系统1来说很正常。
- **思维发散性的"散弹枪"模型**：如同想用散弹猎枪瞄准一个点是不可能的一样，想要让系统1完全执行系统2的命令且不做多余的工作也很难。

## Anti-patterns

- **用平均判断替代总量判断**：系统1能轻松判断平均长度，却无法计算总长度。一个类别的规模及其所包含的实例数量常常被忽略。
- **原型替代数量**：在拯救鸟类实验中，人们反应的对象是原型（一只无助的小鸟的画面），而非实际数量。
- **面部特征替代能力评估**：选民用对候选人面部特征的能力评估替代对其政治能力的复杂判断。
- **思维发散导致的干扰**：在执行一个任务时，系统1会自动运行其他评估程序，干扰主要任务的完成（如听押韵时比较拼写）。

## Key Takeaways

1. 系统1不断监视着大脑内外发生的一切，对当时的情形作出全方位评估——这些"基本判断"在直觉性判断中扮演重要角色。
2. 系统1具备跨维度解读价值观的能力（强度匹配），可以回答"如果山姆的身高和智商一样，那么他究竟有多高"这类问题。
3. 系统1能解决平均问题，却对总量问题束手无策——一个类别的规模及其所包含的实例数量常常被忽略。
4. 仅凭0.1秒的人脸照片评估，约有70%的竞选胜出者正是那些在照片评估中获得较高评价的人。
5. 拯救2000只鸟和拯救200000只鸟的平均捐款几乎没有差别——人们反应的对象是原型，而非数量。
6. 思维的发散性使系统1在执行一个任务时会自动运行其他评估程序，这既是优势也是干扰。
7. 系统1通过原型或一组典型事例来代表不同事物分类，它能解决好平均问题，但对总额变量就束手无策。
