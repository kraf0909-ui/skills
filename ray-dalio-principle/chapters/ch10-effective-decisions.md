# Chapter 10: 学习如何有效决策

## Core Idea
Good decision-making is a learnable skill based on logic, expected value calculation, and systematic analysis. The biggest threat to good decisions is harmful emotion; the best tool is a two-step process of learning first, then deciding.

## Frameworks Introduced
- **Two-Step Decision Process**: Learn first, then decide
  - When to use: All significant decisions
  - How: Gather and analyze all relevant information before forming a conclusion
- **Expected Value Calculation**: Probability × Payoff for each option
  - When to use: Decisions under uncertainty
  - How: Estimate probabilities and outcomes, calculate weighted averages
- **80/20 Rule (Pareto)**: Focus on the vital few, not the trivial many
  - When to use: Prioritization, resource allocation
  - How: Identify the 20% of factors that drive 80% of outcomes
- **Convert Principles to Algorithms**: Systematize decision-making
  - When to use: Repeated decisions with clear criteria
  - How: Write rules, backtest on history, implement in code

## Key Concepts
- **Base rates**: Historical frequencies as starting point for probability estimates
- **By-above and by-below**: Distinguish high-level strategic discussions from tactical details
- **Expected value**: The probability-weighted average of all possible outcomes
- **Decision quality**: Separate the quality of the decision process from the outcome
- **Simplify**: Reduce complexity to essential factors

## Mental Models
- Think of decisions as bets — you're betting on which option has the highest expected value
- Use "by-above / by-below" to keep discussions at the right level of abstraction
- View AI and algorithms as partners that can process more data than you can

## Anti-patterns
- **Deciding before learning**: Forming conclusions, then seeking confirming evidence
- **Analysis paralysis**: Over-analyzing to the point of inaction
- **Outcome bias**: Judging decisions by results rather than process quality
- **Ignoring base rates**: Starting from scratch instead of using historical data
- **Perfect information fallacy**: Waiting for certainty that will never come

## Key Takeaways
1. Decision-making is a two-step process: learn first, then decide
2. Calculate expected value — probability × payoff — for each option
3. Use the 80/20 rule to focus on what matters most
4. Separate decision quality from outcome — good decisions can have bad outcomes
5. Simplify — complexity is the enemy of execution
6. Convert repeatable decisions to algorithms
7. Don't over-rely on AI without deeply understanding it first
